# Samantha Key Design Direction

## Product Promise

Samantha Key is a premium voice translation keyboard for people who want to speak naturally and insert translated text into any iPhone text field. It should feel native, private, fast, and more refined than a generic AI keyboard.

## Visual System

- Core mark: white voice orb paired with a compact keyboard silhouette, black waveform, soft cyan signal edge.
- Backgrounds: light grouped iOS surfaces in the app, translucent keyboard glass in the extension, dark high-contrast marketing moments on the web.
- Accent: cyan for voice capture, soft green for translation ready, black capsule for primary voice actions.
- Typography: system semantic styles only; bold reserved for state, action, and high-value product promises.
- Layout: one primary voice action, visible output language, live translated text, setup guidance, and privacy reassurance.
- Motion: subtle pulse while recording or pending handoff. Respect Reduce Motion.

## Product UI Notes

- The keyboard should look like a purpose-built input surface, not a miniature web app.
- The app should explain the Apple-approved handoff honestly: the keyboard launches the companion app for microphone capture, then returns translated text for insertion.
- Onboarding and paywall must be compact enough for small phones and explicit about monthly pricing.
- Avoid generic gradient cards, fake chat logs, and decorative feature stacks.

## App Store Assets

- Icon must be text-free and legible at 60px.
- Screenshots should show: keyboard microphone, live translated text, app recording handoff, keyboard setup, monthly trial/paywall, and privacy.
- Generated marketing art can support the deck, but the actual app UI must remain truthful.
